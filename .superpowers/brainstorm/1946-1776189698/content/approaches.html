Rebuild Approach

Two ways to structure the Astro project. Both deploy to Netlify, both produce static HTML.

A

Content Collections (Recommended)

Projects, services, testimonials, and FAQs stored as Markdown/MDX files in src/content/. Astro generates pages from them automatically. Adding a new project = adding a new .md file with frontmatter (title, client, stats, images). No code changes needed to add content.

Pros

  • Easy content updates without touching code
  • Type-safe content schemas via Astro
  • Simple to add a CMS later (e.g. Decap CMS)
  • Clean separation of content and layout

Cons

  • Slightly more initial setup
  • Need to learn Markdown frontmatter conventions
B

Hardcoded Pages

Each page is a standalone .astro file with content written directly in the template. Simpler structure, but every content change means editing component files. Projects, services, FAQs all live in the page templates.

Pros

  • Simpler to understand at first glance
  • No Markdown/frontmatter to learn
  • Fewer files initially

Cons

  • Content changes require editing code
  • Harder to add CMS later
  • More repetitive -- each project page is manual
  • Doesn't scale well as content grows
